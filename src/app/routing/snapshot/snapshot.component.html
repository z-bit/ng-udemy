Snapshot Component

If the above ID changes and Route with id will be clicked the according parameter does not change as the child gained the parameter from a snapshot in the constructor: 

this.id = activatedRoute.snapshot.params['id'];

As the component still is displayed the constructor is not executed, no new snapshot. If you close the child with Got it! and call it again the changed parameter will be shown. A new snapshot is created.
